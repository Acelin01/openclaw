## ⚠️ 发送消息没回复

### 可能原因

1. **WebSocket 连接问题**
   - chat-lite 尝试连接 `ws://localhost:18789`
   - 需要确保 OpenClaw Gateway 正在运行

2. **Gateway 配置问题**
   - 检查 Gateway 是否允许 chat-lite 连接
   - 检查 token 是否正确

3. **消息路由问题**
   - 消息可能发送了但没有正确处理
   - 检查 chat-lite 的消息处理逻辑

### 检查步骤

```bash
# 1. 检查 Gateway 是否运行
curl -s http://localhost:18789/status

# 2. 检查 chat-lite 连接状态
# 打开浏览器控制台 (F12)，查看 WebSocket 连接

# 3. 查看 Gateway 日志
tail -f /Users/acelin/Documents/Next/AIGC/openclaw/.openclaw/logs/*.log
```

### 临时解决方案

如果只是需要查看 artifact 组件，可以直接访问测试页面：
- http://localhost:3000/test-requirement-list.html
- http://localhost:3000/test-testcase-artifact.html

这些页面不依赖 Gateway，直接显示硬编码的示例数据。
