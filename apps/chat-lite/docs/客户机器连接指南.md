## 🌐 不同场景的连接方式

### 场景 1：客户机器在同一局域网 (推荐)

**适用情况**：
- 客户机器和你都在 192.168.31.x 网段
- 同一办公室/同一 WiFi

**连接配置**：
```yaml
gateway:
  url: ws://192.168.31.204:60123
  token: openclaw-platform-token-2026
```

**客户机器需要做的处理**：
1. 确认你的电脑 IP 是 192.168.31.204
2. 更新客户机器的配置文件
3. 测试连接：`nc -zv 192.168.31.204 60123`

### 场景 2：客户机器在外部网络

**适用情况**：
- 客户机器在不同城市
- 通过 Internet 连接

**需要配置**：

#### 方案 A：路由器端口转发
1. 登录你的路由器管理界面 (通常 192.168.31.1)
2. 添加端口转发规则：
   - 外部端口：60123
   - 内部 IP：192.168.31.204
   - 内部端口：60123
   - 协议：TCP
3. 客户机器使用你的公网 IP 连接

#### 方案 B：使用内网穿透工具
- ngrok
- frp 公网服务器
- 花生壳

## 🔧 客户机器配置步骤

### 步骤 1：更新配置文件

找到客户机器上的配置文件（通常是 `config.json` 或 `agents.config.json`）

**修改前**：
```json
{
  "gateway": {
    "url": "ws://192.168.31.154:60123",
    "token": "openclaw-platform-token-2026"
  }
}
```

**修改后**：
```json
{
  "gateway": {
    "url": "ws://192.168.31.204:60123",
    "token": "openclaw-platform-token-2026"
  }
}
```

### 步骤 2：重启客户机器服务

```bash
# 根据客户机器的具体服务命令
systemctl restart customer-bot
# 或
docker restart customer-bot
# 或
pm2 restart customer-bot
```

### 步骤 3：验证连接

```bash
# 测试端口连通性
nc -zv 192.168.31.204 60123

# 测试 WebSocket 连接
# 使用 wscat 或其他 WebSocket 客户端
wscat -c ws://192.168.31.204:60123

# 查看客户机器日志
tail -f /var/log/customer-bot.log
```

## 📊 连接测试工具

### 在你的电脑上测试

```bash
# 1. 确认 Docker 服务运行
docker-compose -f docker-compose-uxin.yml ps

# 2. 确认端口监听
netstat -tlnp | grep 60123

# 3. 本地测试
nc -zv 127.0.0.1 60123

# 4. 局域网测试（从你的电脑）
nc -zv 192.168.31.204 60123
```

### 在客户机器上测试

```bash
# 1. Ping 测试
ping 192.168.31.204

# 2. 端口测试
nc -zv 192.168.31.204 60123

# 3. telnet 测试（如果 nc 不可用）
telnet 192.168.31.204 60123
```

## 🔍 故障排查

### 问题 1：连接超时

**可能原因**：
- 防火墙阻止
- 端口未开放
- IP 地址错误

**解决方法**：
```bash
# 在你的电脑上检查防火墙
sudo ufw status
sudo ufw allow 60123/tcp

# macOS 防火墙
# 系统偏好设置 → 安全性与隐私 → 防火墙 → 防火墙选项
# 添加 Docker 到允许列表
```

### 问题 2：连接被拒绝

**可能原因**：
- Docker 容器未运行
- FRP 隧道未建立

**解决方法**：
```bash
# 重启 Docker 服务
cd /Users/mac/Documents/GitHub/openclaw/apps/chat-lite
docker-compose -f docker-compose-uxin.yml restart
```

### 问题 3：客户机器无法 Ping 通

**可能原因**：
- 不在同一网络
- 路由器隔离

**解决方法**：
1. 确认客户机器和你的电脑在同一网段
2. 检查路由器设置（关闭 AP 隔离）
3. 使用有线连接代替 WiFi

## 📝 完整配置示例

### 客户机器配置文件 (config.json)

```json
{
  "bot": {
    "name": "客户机器人",
    "version": "1.0.0"
  },
  "server": {
    "gateway": {
      "url": "ws://192.168.31.204:60123",
      "token": "openclaw-platform-token-2026",
      "reconnect": true,
      "reconnectInterval": 5000
    },
    "timeout": 30000,
    "heartbeat": 30000
  },
  "logging": {
    "level": "info",
    "file": "/var/log/customer-bot.log"
  }
}
```

### 客户机器连接脚本 (connect.sh)

```bash
#!/bin/bash

SERVER_IP="192.168.31.204"
SERVER_PORT="60123"

echo "测试连接到客户机器人服务器..."
echo "服务器：${SERVER_IP}:${SERVER_PORT}"
echo ""

# Ping 测试
echo -n "Ping 测试... "
if ping -c 1 -W 2 ${SERVER_IP} > /dev/null 2>&1; then
    echo "✅ 成功"
else
    echo "❌ 失败"
    exit 1
fi

# 端口测试
echo -n "端口测试... "
if nc -z -w 2 ${SERVER_IP} ${SERVER_PORT} > /dev/null 2>&1; then
    echo "✅ 成功"
else
    echo "❌ 失败"
    exit 1
fi

echo ""
echo "✅ 连接测试通过！"
echo "可以启动客户机器人服务了"
```
