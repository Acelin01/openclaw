# 简历求职工作流

本指南介绍了一种使用 OpenClaw 进行求职申请的**人机协作（Human-in-the-Loop）**工作流。由于许多招聘平台（如 Boss 直聘、LinkedIn）具有严格的反爬虫措施（验证码、扫码登录），完全自动化的“无人值守”流程往往不可靠。相反，我们推荐一种协作方式：由智能体处理重复性任务（简历解析、职位搜索、初步筛选），而你只需处理关键的身份验证步骤。

## 前置条件

1.  **简历文件**：准备一份 OpenClaw 可读取格式（PDF、Markdown 或文本）的简历。
2.  **浏览器技能**：确保 `openclaw.json` 中已启用 `browser` 技能。

## 工作流概览

1.  **简历分析 (Agent)**：智能体读取你的简历，提取关键技能、工作经验，并生成个性化的求职信/打招呼话术。
2.  **浏览器启动 (Agent)**：智能体启动受控的浏览器实例。
3.  **身份验证 (User)**：你在智能体打开的浏览器窗口中手动登录招聘平台（例如扫码）。
4.  **职位搜寻 (Agent)**：智能体接管后续操作，根据你的标准搜索、筛选职位，并发起沟通或申请。

## 分步指南

### 1. 简历分析

首先，让智能体理解你的个人档案。

```bash
# 示例：读取简历并提取关键词
openclaw file read --path "/Users/me/Documents/resume.pdf"
```

_给智能体的提示词：_

> “读取位于 /path/to/resume.pdf 的简历。提取我的前 5 项技术技能、工作年限，并为招聘人员生成一段礼貌的打招呼消息。”

### 2. 启动浏览器并登录

启动 OpenClaw 浏览器会话。

```bash
# 启动浏览器服务
openclaw browser start

# 导航到招聘平台（例如 Boss 直聘）
openclaw browser open "https://www.zhipin.com"
```

**需人工操作**：
此时，浏览器窗口将会打开。**请手动扫描二维码**或输入账号密码登录。登录成功后，你可以关闭登录弹窗，但请保持浏览器窗口开启。

### 3. 自动化搜索与申请

登录完成后，指示智能体开始搜索和处理职位。

_给智能体的提示词：_

> “我已经登录了。现在搜索‘上海’的‘资深前端工程师’职位。筛选描述中包含‘AI’或‘SaaS’的公司。对于每个匹配项，点击职位卡片，检查是否要求‘React’，如果是，点击‘立即沟通’并发送我的打招呼消息。”

智能体将使用浏览器自动化工具（点击、输入、滚动、阅读）在网站上进行导航操作。

### 成功贴士

- **会话持久化**：OpenClaw 的浏览器配置文件会保存 Cookies。通常你每隔几天才需要登录一次。
- **频率限制**：招聘平台可能会封锁频繁的爬取行为。请指示智能体“在操作之间暂停 3-5 秒”以模仿人类行为。
- **可视化验证**：你可以实时在浏览器窗口中观察智能体的操作。如果它卡住了（例如出现了验证码），你可以立即介入处理。
