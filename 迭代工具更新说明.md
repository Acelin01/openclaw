## 📝 使用方式

### 创建需求并关联到迭代

**方式 1: 创建时直接关联**
```javascript
{
  "action": "requirement_create",
  "params": {
    "title": "用户登录功能",
    "iteration_id": "iter-xxx",  // 直接指定迭代
    ...
  }
}
```

**方式 2: 创建后批量规划**
```javascript
// 先创建需求（不指定迭代）
{
  "action": "requirement_create",
  "params": {
    "title": "用户登录功能",
    ...
  }
}

// 然后批量规划到迭代
{
  "action": "iteration_plan",
  "params": {
    "iteration_id": "iter-xxx",
    "requirement_ids": ["req-1", "req-2"],
    "task_ids": ["task-1"]
  }
}
```

## 🎯 工作项概念说明

### 什么是工作项？

**工作项** 是需求、任务、缺陷的总称，用于迭代管理中的统一统计和展示。

```
迭代 (Iteration)
└── 工作项 (Work Items)
    ├── 需求 (Requirements)
    ├── 任务 (Tasks)
    └── 缺陷 (Defects)
```

### 为什么移除 `iteration_add_workitem`？

1. **工作项不是独立实体** - 它只是需求/任务/缺陷的统称
2. **应该直接操作具体类型** - 创建需求/任务/缺陷
3. **批量操作用 `iteration_plan`** - 统一规划更清晰
